# 공시 숫자 표 파싱 개선 계획

## Summary
현재 프로그램은 `국민연금공단`이 들어간 공시를 찾는 데는 성공했지만, 숫자 표를 읽는 단계에서 실패하고 있습니다. 원인은 4월 20일 공시가 기존에 예상한 "컬럼형 보유주식 표"가 아니라, DART 목차별 viewer 화면에 나뉘어 있는 `임원ㆍ주요주주 특정증권등 소유상황보고서` 형식이기 때문입니다.

새 파서는 보고서 종류별로 읽는 방식을 나눕니다. 기존 `주식등의대량보유상황보고서` 파서는 유지하고, `임원ㆍ주요주주특정증권등소유상황보고서` 전용 파서를 추가합니다.

## Key Changes
- DART `main.do`에서 목차 정보를 읽고, 각 목차의 `viewer.do` 본문을 가져옵니다.
- `1. 발행회사에 관한 사항`에서 회사명, 회사코드, 발행주식 총수를 읽습니다.
- `2. 보고자에 관한 사항`에서 보고구분과 보고자 정보를 읽습니다.
- `3. 특정증권등의 소유상황`에서 `직전보고서`, `이번보고서`, `증 감` 행을 읽습니다.
- `이번보고서`의 주권 주식수와 비율을 현재 보유 수량과 지분율로 저장합니다.
- `증 감`의 주권 주식수와 비율을 변동 수량과 변동 지분율로 저장합니다.
- 기존 `HoldingEvent` 모델에 우선 맞춰 저장합니다.

## Implementation Plan
- `src/collect/probe.py`의 목차 추출 로직을 재사용해 `receipt_no -> viewer sections` 함수로 정리합니다.
- `src/parse/report_parser.py`를 보고서 종류별 라우터로 바꿉니다.
- `임원ㆍ주요주주` 또는 `특정증권` 보고서는 새 전용 파서를 사용합니다.
- 기존 `주식등의대량보유상황보고서`는 기존 컬럼형 표 파서를 유지합니다.
- 파싱 실패 시 어느 목차와 필드에서 실패했는지 `parse_error`에 남깁니다.

- v1에서는 `이번보고서` 행의 `주권 주식수`와 `주권 비율`을 현재 보유 상태로 사용합니다.
- `특정증권등` 전체 수량보다 `주권` 수량을 우선 사용합니다.
- 거래별 세부 내역은 v1 범위에서 제외합니다.
- 구조가 다른 예외 공시는 `parse_error`에 남기고, 샘플이 쌓이면 파서를 확장합니다.
